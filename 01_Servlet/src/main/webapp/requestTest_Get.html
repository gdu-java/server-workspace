Get방식으로 요청 후 응답페이지 받아보기

특징1. GET방식으로 요청시 URL의 Header영역에 데이터들을 파라미터로 포함시켜 요청함.
=> 사용자가 입력한 값(데이터)들이 URL에 노출됨.
=> 보안유지가 어려움.
=> 즉, 로그인이나 회원가입 같은 경우 GET방식은 부적합함.

특징2. Header영역은 전송하는 길이에 제한이 있음.
=> 방대한 데이터를 담았을 경우 초과된 데이터는 절단되어 전달됨.
=> 즉, 게시판 등의 내용을 작성할 경우 GET방식은 부적합함.

특징3. 장점은 URL에 데이터가 노출되기 때문에 즐겨찾기(북마크, URL을 저장해놨다가 재요청)에 유용. => 즉, 검색 기능 같은 경우 GET방식이 적합함.

개인정보 입력

  • 이름 :
  • 성별 : 남자 여자
  • 나이 :
  • 거주지 :
  • 키 :
  • 좋아하는 음식(모두 고르시오.) 한식 중식 일식 양식 분식
